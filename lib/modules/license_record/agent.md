# license_record 模块说明

## 模块定位

`license_record` 是授权管理工具中的核心业务模块。

负责：
- 新建授权
- 生成授权码
- 保存授权记录
- 查询授权记录
- 导出授权记录
- 导入已有授权记录
- 导入名单后批量生成授权码
- 维护授权有效期天数与首次激活截止日期

## 分层职责

- `data/`：授权记录实体与 repository
- `domain/`：发码、记录管理、导出、导入逻辑
- `presentation/`：总览、表单、列表、详情

## 批量导入规则

- 所有正式导入都必须通过系统文件选择器选择文件
- 不得将手动输入文件路径作为正式导入交互入口长期保留
- 历史授权记录导入必须以 `licenseId` 作为唯一匹配键
- 历史授权记录导入模板必须包含最后一列 `操作标记`
- 历史授权记录导入模板统一使用 `.xlsx`
- 历史授权记录导入文件名称允许扩写，但必须包含系统下载模板的名称
- `操作标记` 允许值仅为 `I / U / D`
- 单次导入最多支持 `1000` 条数据
- 导入失败结果必须完整展示失败原因
- 导入列数不得多于模板要求字段个数
- 必须提供清晰的规则说明展示位置
- 必须提供配套模板下载入口
- 历史授权记录导入时：
  - `I`：`licenseId` 必须不存在
  - `U`：`licenseId` 必须已存在
  - `D`：`licenseId` 必须已存在
- 历史授权记录导出时，最后一列必须补 `操作标记`，默认值写为 `I`
- 待发码名单导入也必须包含最后一列 `操作标记`
- 待发码名单导入模板统一使用 `.xlsx`
- 待发码名单导入文件名称允许扩写，但必须包含系统下载模板的名称
- 待发码名单导入当前阶段仅允许 `I`
- 原因：该模板由系统统一生成新的 `licenseId`，当前没有稳定业务主键支撑 `U / D`
- 批量生码模板导出时，最后一列必须补 `操作标记`，默认值写为 `I`
- 批量生码与历史记录导入必须使用不同模板，不得混用
- 导入或导出结果如展示文件路径或目录信息，提示默认显示时长统一为 `5` 秒
- 涉及文件或目录位置提示时，应提供“打开目录”动作
